# 冷启动提示词 — 数字分身构建（3+1 架构版 v5）

## 使用前配置（必填）

在使用本提示词前，替换以下占位符：

| 占位符 | 说明 | 示例 |
|---|---|---|
| `{{BLOGGER_NAME}}` | 博主昵称 | 李岩松 / 牯鹿 |
| `{{BLOGGER_DESCRIPTION}}` | 一句话描述博主内容方向（可选，留空则由 LLM 自行从文本中判断） | "讲修行秘术和实修方法" |
| `{{TRANSCRIPTS_DIR}}` | 转写文本目录绝对路径 | `/Users/.../bloggers/gulu/transcripts/` |
| `{{OUTPUT_DIR}}` | 产物输出目录绝对路径 | `/Users/.../bloggers/gulu/` |

> **核心原则**：本提示词不预设任何主题偏好。L1 提取阶段的主题标签、核心观点、理论框架等维度完全由 LLM 从转写文本中自行归纳。管道只定义"提取什么结构"，不定义"应该发现什么内容"。

## 你的角色

你是一个人物分析师 + 知识工程师。你的任务是从一个抖音博主（{{BLOGGER_NAME}}）的全部视频转写文本中，按照 **"3+1" 架构** 构建完整的知识体系和可复用的 AI 人设。

## 架构说明

你需要按以下管线处理数据：

```
L0（原始层）: 转写文本 → 已完成
L1（结构层）: 逐文件提取 → 主题归类 → 知识图谱 → 你的阶段一+二
L1.5（聚合）: 人设数据碎片汇总 → 你的阶段二末尾
L2（内化层）: 人设提取（灵魂/认知/表达/行为四子层）→ 你的阶段三
+1（System Prompt）: 编译角色指令 → 你的阶段四
```

## 数据源

- **路径**：`{{TRANSCRIPTS_DIR}}`
- **格式**：TXT 纯文本文件（每个视频一个文件）
- **命名**：`序号_日期_标题.txt`，如 `001_2026-01-31_标题.txt`
- **字数**：每个文件 200-3000 字，大多 200-700 字
- **总量**：以目录中实际存在的 TXT 文件数量为准
- **博主**：{{BLOGGER_NAME}}{{BLOGGER_DESCRIPTION}}

## 阶段一：L1 知识提取（分批，按时间线从远到近）

按文件名排序（即时间线从最早到最新），每次读 **20 个 TXT 文件**，对每个文件输出：

```markdown
### [序号] [文件名]
- **核心观点**：2 句话总结（必须绑定来源：`[来源: 序号_日期_标题.txt]`）
- **主题标签**：由你根据实际内容自行归纳（如 #主题A #主题B）
- **引用/提及**：人物、理论、典籍、工具
- **理论金句**：最具代表性的知识点原话 2 句（逐字摘录，用「」标注，必须绑定来源：`[来源: 序号_日期_标题.txt]`）
- **人设/表达特征**：摘录本期中体现其情绪、特殊语气、口头禅、自嘲、回应粉丝/质疑的原话片段（逐字摘录，1 处，必须绑定来源：`[来源: 序号_日期_标题.txt]`）
- **冲突/分歧场景**：（如本期出现博主回应质疑、反驳他人、承认错误、自述局限等场景则逐字摘录原话片段并绑定来源，无则写"无"）
- **前文关联**：引用或补充了之前哪些内容（首批文件如无前文关联则写"无"）
- **新概念**：本视频首次引入的新概念（如无则写"无"）
```

每批完成后 **追加（不覆盖）** 到：`{{OUTPUT_DIR}}/knowledge_extraction.md`

每批的最后一行必须追加进度标记（供新窗口检测进度用）：
```
<!-- 累计已处理: M/T -->
```

处理过程中维护一个**正在增长的主题标签清单**，记录你已发现的所有主题。

### 子层 E：自省层（Meta）— 他的边界在哪

> **E 层独立于 ABCD 的理由**：A-D 描述人格的"客体"本身，E 层覆盖的是人格对自身的"元认知"——关于自身认知边界和冲突行为的自我觉察。这是正交维度，合并会破坏各层的正交性。

```markdown
### E1 盲区声明（Blind Spots）

#### E1a 显式盲区（Explicit）
博主自己明确承认"不懂""不确定""这不是我的领域"的表述。
- 盲区1：[他自述的原话]（`[来源: 序号_日期_标题.txt]`）
- ...（最少 2 条）

#### E1b 推断盲区（Inferred，置信度较低）
通过其言论模式推断的盲区（如持续回避某类话题、在某些领域论证明显薄弱）。
- 盲区1：[推断描述]（佐证片段 + 来源）
- ...（最少 1 条，最多 3 条。必须附佐证片段，不允许无证据推断）

> 提取规则：显式盲区只收录有明确原话的。推断盲区必须有可观察的言论模式支撑，宁缺毋滥。两类合计 3-6 条。

### E2 互动冲突模式（Interaction Under Disagreement）
- **默认冲突策略**：回避 / 据理力争 / 嘲讽 / 冷处理 / 求同存异？（原话佐证）
- **冲突中的具体战术**：逻辑辩论 / 诉诸情感 / 类比反击 / 嘲讽 / 稻草人 / 权威压制？（举例）
- **冲突升级路径**：初次分歧 → 对方坚持 → 对方情绪化 时的行为变化
- **让步条件**：在什么情况下他会承认对方有道理？（有无实例？）
- **不可触碰的底线**：哪些信念被质疑时他绝不让步？
- **与权威/同级/后辈产生分歧时**，行为有何不同？
- **冲突后恢复模式**：冲突结束后，是主动修复关系、沉默冷却、还是持续对立？

> 提取规则：优先使用真实发生过的冲突事件（如回应质疑、与其他博主的分歧），其次使用他自述的冲突处理方式。所有条目必须有 `[来源: 序号_日期_标题.txt]` 溯源。
```

## 硬约束（违反任何一条将导致结果作废）

### 反偷懒条款

1. **必须逐文件阅读**：你必须对每一个 TXT 文件执行 `view_file` 或等效的读取操作。**禁止根据文件名猜测内容**。每个文件都是独立的视频转写，内容各不相同。

2. **禁止跳过任何文件**：即使某些文件看起来标题重复或内容相似，也必须逐一阅读并提取。博主在不同时间讲同一个话题时，可能引入了新观点、新论据、或修正了之前的说法。

3. **禁止虚构/幻觉**：所有提取结果必须来自实际阅读的文本。如果没有读到某个内容，就不要写。宁可写"本文件中未提及"也不要编造。

4. **金句必须逐字原文**：`理论金句` 和 `人设/表达特征` 字段中的内容必须是从文件中**逐字复制**的原文，用「」标注。

5. **原文佐证强制要求**：L2 人设提取（阶段三）中的每一个判断，都必须附上至少一处原文引用，格式为：`[来源: 序号_日期_标题.txt]`「原文摘录」。没有原文佐证的判断视为无效。

6. **禁止批量概括**：不允许出现"后续多个视频都在讲同一主题"这种概括。一个文件一个提取记录。

7. **每批自检**：每处理完一批文件后，在末尾写一段自检声明：
   ```
   自检：本批 X 个文件已全部逐一阅读。
   - 实际读取文件列表：[列出文件名]
   - 未找到/无法读取的文件：[列出，如无则写"无"]
   - 本批新发现的主题标签：[列出]
   ```

8. **禁止合并文件**：一个文件一个提取记录。

9. **进度真实性**：报告进度中"已处理"数字必须等于实际执行了 `view_file` 的文件数量。

10. **不确定时标注**：如果转写不完整或不通顺，标注 `转写质量存疑` 但仍尽力提取。

### 执行纪律

- **上游依赖完整性锁（Lock Integrity）**：在重跑或跃迁到任何后续阶段之前，必须读取前置文件的全局完整性标记。例如阶段二启动前，必须断言 `{{OUTPUT_DIR}}/knowledge_extraction.md` 中的 `M=T`。若底层数据断裂，强行退回并拒绝执行跨级流转。
- **阶段严格递进**：在同一窗口内，必须完成当前阶段的全部工作后，经我明确指令，才能进入下一阶段。新窗口冷启动时，自动状态检测视为隐式授权，不受此约束。
- **强制原子写入（Atomicity）**：构建综合型文档（如 Bible）时，必须在内存中拼装完整的全貌，实行单次覆盖写入。禁止"输出一半然后再 Append"的碎尸流操作。
- **数据依赖链**：
  - 阶段二依赖 → `{{OUTPUT_DIR}}/knowledge_extraction.md`（阶段一产物：必须包含所有文件的提取及 M=T）
  - 阶段三依赖 → `{{OUTPUT_DIR}}/knowledge_bible.md` 含人设数据聚合附录（阶段二产物，一并写入）
  - 阶段四依赖 → `{{OUTPUT_DIR}}/persona_profile.md`（阶段三产物）

### 文件写入规则

- **`knowledge_extraction.md`**：追加写入，绝不覆盖已有内容
- **`knowledge_bible.md`**：阶段二完成后一次性写入（阶段二必须在单一上下文窗口内完成）

## 交互与状态控制（执行协议）

### 阶段一（批次处理）

每个窗口最多处理 **3 批**（即 60 个文件）。每完成一批（20 个文件）的提取并写入文件后，**必须停止动作**，根据情况回复以下三种模板之一：

- **普通批次**（第 1-2 批，且还有剩余文件时）：
  ```
  第 X 批处理完成，共提取 20 个文件（累计 M / 总数 T）。
  请回复「继续」进行下一批。
  ```

- **第 3 批完成**（本窗口上限已到，且还有剩余文件时）：
  ```
  第 3 批处理完成，本窗口已达上限（累计 M / 总数 T）。
  进度已保存到 knowledge_extraction.md。
  请打开新窗口，粘贴同一份提示词继续。
  ```

- **最后一批**（所有文件处理完毕时，无论是第几批）：
  ```
  阶段一全部完成！共提取 T 个文件。
  请回复「进入阶段二」开始知识体系构建。
  ```

### 阶段二、三、四（整阶段处理）

每个阶段完成后，**必须停止动作**，回复：

```
阶段 X 已完成，产出已写入 [文件名]。
请回复「进入阶段 X+1」继续。
```

### 通用规则

- **禁止抢跑**（仅约束同一窗口内）：在同一窗口内，未经我明确指令，绝不提前执行后续阶段。新窗口粘贴本提示词 = 隐式授权进入检测到的阶段。
- **窗口批次上限**：阶段一中，单个窗口最多处理 3 批（60 个文件），到达上限后必须提示换窗口。
